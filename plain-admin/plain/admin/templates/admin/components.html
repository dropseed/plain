The design system has three layers, loaded in order:

  1. Tailwind v4 — utility classes
  2. basecoat.css — neutral component primitives (buttons, cards, inputs, …)
  3. admin.css — Plain's brand palette and admin-only chrome

To re-skin the admin, add your own stylesheet after these and override the tokens. Here's a minimal example that swaps the primary action color to indigo: 

/* app/static/admin-overrides.css */
:root {
  --primary: #4f46e5;        /* indigo-600 */
  --primary-foreground: white;
  --ring: #4f46e5;
}
.dark {
  --primary: #818cf8;        /* indigo-400 */
  --primary-foreground: #1e1b4b;
}

Then load it in your admin/base.html override after the admin's own CSS. Every .btn-primary, focus ring, and link-color in the admin will update.

The toggle stores the user's choice in localStorage under plain-admin-theme. Resolution happens before paint via an inline init script in base.html, so there's no flash on page load. Custom CSS only needs the standard Tailwind dark: variant — the .dark class is on the <html> element when dark mode is active.
